JesusChristGames Site Plan

WDD 131 • W05 Project: Website Planning Document

Site Name

JesusChristGames — a collection of simple, family friendly scripture based games for youth and Sunday activities. The name is clear and easy to remember and fits the audience we serve.

Optional domain availability: jesuschristgames.org or jesuschristgames.app

Site Purpose

The site provides a hub for gospel themed mini games. Visitors can pick a game, read quick instructions, and play on mobile or desktop. Leaders and parents can access a lightweight resources page with printable rules and scripture references.

  • Games catalog with filters by age range and play time
  • Individual game pages with rules, start buttons, and scripture references
  • Resources page with printable PDFs and links to talks and lessons
  • Feedback form to request new game ideas

Scenarios

  • Which quick game can I run for 10 minutes with teens before class starts
  • Where do I find a scripture reference list to match the trivia questions

Color Schema

The plan uses two main brand colors with neutrals. Use the primary teal for headings, navigation, and links. Use the amber for buttons and badges. Use neutrals for body text and panels.

Primary teal #0f766e for headings, nav, links
Secondary amber #f59e0b for buttons and highlights
Body text #0b132b
Panel background #f8fafc
Apply this same schema in the final site. Keep contrast above 4.5:1. Validate with DevTools CSS Overview.

Typography

  • Playfair Display for headings h1–h3
  • Inter for body copy, navigation, buttons, and form labels

Both families are loaded from Google Fonts in this document. Use the same import on the project pages.

CSS and components

  • Create a shared CSS file /project/styles/site.css and reuse the variables above
  • Buttons use the amber as background and white text
  • Links use the teal color and underline on hover

Testing and validation

  1. Run Lighthouse for mobile and desktop. Target 90 or above in all four categories
  2. Use DevTools CSS Overview to check contrast. Adjust if any item fails
  3. Validate HTML and CSS with the W3C validators
